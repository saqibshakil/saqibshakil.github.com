Hover over the folder or files to get info on the usage of these files in the tooltip. Below I will explain the usage of individual files placed in the module folder

Modules Folder

This folder contains individual modules. Ideally each module would be contained in its own folder.

GMC/Connect Folder

These individual folders are just samples with no worthy implementation. But serves as a bassis for future development.

Main.ts

Primarily this file would contain the initialization logic of the modules and would reference other dependencies of the modules. Ideally a module should have a minimal number of files. but the architecture currently provides for any number of files. In production a grunt or brunch type build system could be utilized to build a singular js file per module where as the development could be done in any number of seperate files.

Models.ts

Contains any and all Models used by the current module.

Views.ts

Contains any number of Views used by the module. Templates are also referenced by this file. Currently I have referenced each template individually but in production templates should be merged in to singular files and referenced thereafter.

Router.ts

This is one of the most crucial part of the module as it contains the implementation of the router. Whenever a modules is requested by the user by using #ABC\* on the url. The module must have a Router name class in its namespace because this is used to bind the subrouter to the modules url.

ViewModels.ts

This file contains the viewmodels used by the module view models are a part of the MVVM model. MVVM is costly and resource intensive so I opt to use it only when there is a requirement for 2 way binding and some business involved for handling the DOM More on MVVM comming soon.
